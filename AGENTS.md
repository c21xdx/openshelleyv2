# AGENTS.md - AI Agent 工作指南

## 核心原则

**优先级：自动化 > 结果质量 > Token 消耗**

1. **直接做，不要问** - 需求明确就直接开始，不要反复确认
2. **自己调试到能跑** - 写完代码自己测试，遇到错误自己修，不要把错误抛给我
3. **主动验证结果** - 用截图、运行测试等方式确认功能正确
4. **一次做对** - 宁可多花时间做对，不要来回返工

---

## 工作方式

### 开发流程
```
需求 → 写代码 → 运行测试 → 发现问题 → 修复 → 再测试 → 循环直到成功 → 告诉我结果
```

### Web/UI 开发
```
1. 写代码
2. 启动服务 (后台运行)
3. browser_navigate 打开页面
4. browser_take_screenshot 截图查看
5. 有问题就改，改完刷新再截图
6. 循环直到满意
7. 给我看最终截图 + 访问地址
```

### API 开发
```
1. 写代码
2. 启动服务
3. curl 测试各个接口
4. 确认响应正确
5. 告诉我怎么用
```

---

## 调试原则

### 截图
- **主动截图** - 不确定效果就截图看
- **改完截图** - 修改后截图确认
- **问题截图** - 遇到 UI 问题先截图分析

### 错误处理
- **自己修** - 报错了自己看日志、改代码、重试
- **不要放弃** - 一个方案不行就换一个
- **不要问我** - 除非试了很多方法都不行

### 测试
- **必须测试** - 代码写完必须运行验证
- **真实测试** - 不要假装测过，要真的跑
- **完整测试** - 主要功能都要测到

---

## 代码风格

- 简单直接，不要过度设计
- 中文注释
- 用常见的库和方法
- 错误信息要清晰

## 语言选择

- Go, Python, Node.js, Bash 都可以
- 选最适合任务的
- 脚本优先 Bash 或 Python

---

## 完成后

告诉我：
1. **怎么运行** - 启动命令、访问地址
2. **做了什么** - 简要说明，不用贴完整代码
3. **效果截图** - Web 项目给我看截图

---

## 忽略的目录

不要读取这些目录的内容：
- `node_modules/`, `vendor/`, `venv/`, `__pycache__/`
- `dist/`, `build/`, `.next/`, `.nuxt/`
- `.git/objects/`, `.git/logs/`
- `*.db`, `*.sqlite` 文件内容
- `*.lock` 文件内容
